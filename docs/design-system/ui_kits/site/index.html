Arquitetura

O DS é dividido em primitivos (tokens canônicos da marca) + kits (vocabulário por contexto). Cada produto importa os primitivos e adiciona seu kit.

ds/tokens.css ← primitivos: cor, tipo, radius, brand ├─ ds/kits/site.css ← marketing (este projeto) ├─ ds/kits/hub.css ← Hub SaaS (outro projeto, importa tokens daqui) ├─ ds/kits/saude.css ← Simplafy Saúde (futuro) └─ ds/kits/seguros.css ← Simplafy Seguros (futuro)

Regras de ouro: não inventar nova cor de marca em um kit; mudar token = bumpar versão; kits podem adicionar vocabulário próprio (ex: --hub-shell-w), nunca contradizer um primitivo.

Layout · container + section padding

Container max-width 1200px. Section padding tem 3 alturas: tight (56), normal (88), loose (120).

--site-container-max : 1200px  ·  --site-container-pad : clamp(20px, 4vw, 40px)
--site-section-tight : 56px  ·  --site-section-normal : 88px  ·  --site-section-loose : 120px
mobile (≤900px) : 64px  ·  mobile-tight (≤560px) : 48px
